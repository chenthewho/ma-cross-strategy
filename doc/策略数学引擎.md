# 策略数学引擎 — golden_cross 动态均衡策略

**文档目标:** 定义 Step() 函数的完整输入输出契约，以及内部各层的计算逻辑。不含系统架构或进化引擎实现细节。

---

## 0. 引擎身份

**策略 ID:** `golden_cross`

**本质:** 纯函数无状态转换器。唯一入口 `Step(StrategyInput) -> StrategyOutput`。

**铁律复述:**
1. 回测与实盘调用同一个 `Step()` 实现，内部无 `if isBacktest` 分支
2. `Step()` 内部禁止定时器、网络请求、数据库读写、任何文件 I/O
3. `Step()` 只在 SaaS 侧执行
4. 价格计算无量纲化（对数收益率/比率）

---

## 1. 资产结构三态（Portfolio State）

### 1.1 仓位语义

| 仓位类型 | 变量名 | 语义 | 操作权限 |
|---------|--------|------|---------|
| 宏观底仓 | DeadHold | 长期持有，只进不出 | 宏观引擎买入，老化后可释放 |
| 微观浮动仓 | FloatHold | 活跃交易仓位 | 微观引擎可买可卖 |
| 冷封存 | ColdSealedHold | 永不释放的底仓 | 任何操作不可触及 |

### 1.2 核心公式

```
TotalEquity    = CNYBalance + DeadHold + FloatHold + ColdSealedHold
                （所有仓位按当前价格估值）

ReserveFloor   = TotalEquity × micro_reserve_pct
                （微观层资金保留底线，防止微观引擎耗尽全部资金）

SpendableCNY   = max(0, CNYBalance - ReserveFloor)
                （微观引擎可用于买入的 CNY 上限）

CurrentMicroWeight = FloatHold × CurrentPrice / TotalEquity
                （当前微观仓位权重，∈ [0, 1]）
```

### 1.3 micro_reserve_pct

- 含义：微观层资金保留比例，确保宏观引擎始终有资金可用
- 类型：可进化参数（染色体）
- 默认值：0.25
- 边界：[0.05, 0.50]

## 2. 市场状态感知层

### 2.1 状态枚举

| 状态 | 标识 | 含义 | 宏观行为 | 微观行为 |
|------|------|------|---------|---------|
| Quiet（安静） | `quiet` | 低波动横盘 | 按基础节奏 DCA | 粉尘订单归零 |
| Bull（牛市） | `bull` | 上升趋势 | 加速 DCA | BetaMultiplier=1.0 |
| Bear（熊市） | `bear` | 下降趋势 | 减速 DCA（等待更好价格） | BetaMultiplier=1.5（加速响应） |
| Panic（恐慌） | `panic` | 极端下跌 | 暂停 DCA（观望） | BetaMultiplier=2.0（最激进） |

### 2.2 判断逻辑

**第一步：计算指标**

```
EMA_short  = EMA(close, 21)    // 短周期均线
EMA_long   = EMA(close, 55)    // 长周期均线
Volatility = StdDev(log_returns, 21)  // 对数收益率标准差（波动率）
VolMedian  = 滚动历史波动率中位数（112 根 bar）
VolRatio   = clamp(Volatility / VolMedian, 0.5, 4.0)
```

**第二步：状态判定**

```
if VolRatio < 0.7:
    → Quiet（低波动横盘）
elif EMA_short > EMA_long:
    if VolRatio > 2.5:
        → Panic（高波动 + 上升 = 极端情况，极少触发）
    else:
        → Bull（上升趋势）
else:  // EMA_short <= EMA_long
    if VolRatio > 2.0:
        → Panic（高波动 + 下跌 = 恐慌）
    else:
        → Bear（下降趋势）
```

### 2.3 输出结构体

```go
MarketState {
    State                   string   // "quiet" | "bull" | "bear" | "panic"
    TimeDilationMultiplier  float64  // 宏观引擎时间窗口缩放
    BetaMultiplier          float64  // 微观 Sigmoid 激进系数缩放
    IsQuiet                 bool     // 安静态标志，true 时微观粉尘归零
}
```

各状态输出值：

| State | TimeDilation | BetaMultiplier | IsQuiet |
|-------|-------------|----------------|---------|
| quiet | 1.0 | 1.0 | true |
| bull | 1.5 | 1.0 | false |
| bear | 0.5 | 1.5 | false |
| panic | 0.0 | 2.0 | false |

## 3. 信号与目标函数框架

Signal 由三个无量纲特征线性合成：

```
X1 = (Price - EMA_short) / (EMA_short × Volatility)
     // 价格偏离度（偏离短均多少个波动率）

X2 = (EMA_short - EMA_long) / (EMA_long × Volatility)
     // 均线发散度（短均相对长均的偏离）

X3 = (Price - Ref(Price, 5)) / (Price × Volatility)
     // 5 期动量（近期涨跌幅相对波动率）

Signal = a × X1 + b × X2 + c × X3
```

其中：
- a（偏离权重）：正值 = 均值回归（偏离后反向操作），负值 = 趋势跟踪
- b（发散权重）：正值 = 发散后押注收敛，负值 = 趋势延续
- c（动量权重）：正值 = 反转，负值 = 追涨杀跌

Signal > 0 → 倾向减仓（看空），Signal < 0 → 倾向加仓（看多）。

## 4. 宏观引擎

### 4.1 设计哲学

宏观引擎负责长期建仓节奏。核心逻辑：**基于市场状态的时间膨胀 DCA + 价格偏离加速**。

### 4.2 输入

```go
MacroDecisionInput {
    TotalEquity          float64
    SpendableCNY         float64
    CurrentPrice         float64
    DeadHold             float64      // 当前宏观底仓（份额）
    DeadHoldValue        float64      // 当前宏观底仓（CNY 估值）
    MonthlyInject        float64      // 月度注资额（来自 SpawnPoint）
    TimeDilation         float64      // 来自市场状态感知
    PriceDeviation       float64      // (Price - EMA_long) / EMA_long
    DaysSinceLastMacro   int          // 距上次宏观操作的天数
    MacroIntervalDays    int          // 基础 DCA 间隔天数（默认 30，来自染色体）
    MacroAccelThreshold  float64      // 价格偏离触发加速的阈值（来自染色体）
    MacroAccelMultiplier float64      // 加速倍率（来自染色体）
}
```

### 4.3 决策逻辑

```
基础判断：DaysSinceLastMacro >= MacroIntervalDays × TimeDilation
    → 触发基础 DCA 订单，金额 = MonthlyInject

加速判断：PriceDeviation < -MacroAccelThreshold（价格明显低于长均）
    → 额外加仓，金额 = MonthlyInject × MacroAccelMultiplier
    → 加速后重置 DaysSinceLastMacro 计数器

死线兜底：SpendableCNY > MonthlyInject × 3（资金积压超过 3 个月注资额）
    → 强制 DCA，金额 = min(SpendableCNY, MonthlyInject × 2)
    （防止资金长期闲置）

最终订单：OrderCNY = clamp(订单金额, 100, SpendableCNY)
    < 100 CNY 不执行
```

### 4.4 铁律

- 只产出 BUY 意图，永不产出 SELL
- 订单金额与 SpendableCNY 做 clamp
- < 100 CNY 不执行

## 5. 微观引擎（Sigmoid 动态天平）

完全按 plan.md 第二章规格实现，这里仅做策略特有参数补充。

### 5.1 不可进化常量

```
MicroSignalEMABars     = 21    // Signal 的 EMA 窗长
MicroSignalStdDevBars  = 21    // 波动率窗长
MicroVolRatioShortBars = 16    // 楔形过滤短窗口
MicroVolRatioLongBars  = 112   // 楔形过滤长窗口
MinOrderCNY            = 100   // 最小订单金额（粉尘拦截）
```

### 5.2 信号计算（策略特有）

```
// 第一步：准备数据
ema = EMA(closes, MicroSignalEMABars)
sigma = max(StdDev(log_returns, MicroSignalStdDevBars), sigma_floor)

// 第二步：计算无量纲信号（三因子合成）
X1 = (currentPrice - ema) / (ema × sigma)           // 价格偏离度
X2 = (EMA_short - EMA_long) / (EMA_long × sigma)    // 均线发散度
X3 = (currentPrice - closes[-5]) / (currentPrice × sigma) // 5期动量

Signal = a × X1 + b × X2 + c × X3
```

### 5.3 Sigmoid 目标权重（通用）

```
EffectiveBeta  = max(0.01, beta × BetaMultiplier)
InventoryBias  = clamp(CurrentMicroWeight, 0, 1) - 0.5
Exponent       = EffectiveBeta × Signal + gamma × InventoryBias
TargetWeight   = 1 / (1 + exp(Exponent)), clamp(0, 1)
```

### 5.4 订单生成

```
DeltaWeight    = TargetWeight - CurrentMicroWeight
TheoreticalCNY = DeltaWeight × TotalEquity

// 楔形过滤
if |TheoreticalCNY| >= MinOrderCNY:
    OrderCNY = TheoreticalCNY  // 直接下单
elif IsQuiet:
    OrderCNY = 0               // 安静态粉尘归零
elif |DeltaWeight| >= 0.02 or VolatilityRatio >= 1.5:
    OrderCNY = sign(TheoreticalCNY) × MinOrderCNY  // 楔形突破，强制最小订单
else:
    OrderCNY = 0               // 粉尘归零
```

## 6. DeadHold 释放规则

### 6.1 软释放（Soft Release）

**触发条件：** DeadHold lot 老化超过 `SoftReleaseMonths`（默认 6，来自染色体），且 `DeadHold > DeadHoldTarget × TotalEquity / Price`（超过目标底仓比例）。

```
可释放份额 = min(
    DeadHold - DeadHoldTarget × TotalEquity / Price,
    DeadHoldAged,
    DeadHold × MaxSoftReleasePct  // 来自染色体，默认 0.30
)
```

释放后：对应的 SpotLot 从 `DEAD_STACK` 转为 `FLOATING`，保留原始成本价。

### 6.2 硬释放（Hard Release）

**触发条件：** 微观引擎产出 SELL 意图，但 FloatHold 不足，且 DeadHold（非 ColdSealed）有可释放份额。

```
缺口 = 卖出所需份额 - FloatHold
硬释放份额 = min(缺口, DeadHold - ColdSealedDeadHold)
```

释放后转为 `FLOATING` 并立即卖出。

### 6.3 铁律

- ColdSealedHold 标记的 lot 任何情况下不可释放
- 软释放只在 SaaS 侧更新账本，不向 Agent 下发指令（仅写审计日志）
- 硬释放在同一 tick 内完成（释放 → 卖出），合并为一条 SELL TradeCommand

## 7. 可进化参数契约（Chromosome）

### 7.1 染色体字段清单

| 字段名 | 类型 | 语义 | 默认值 | 边界 |
|--------|------|------|--------|------|
| `a` | float64 | 价格偏离权重（X1系数） | 0.0 | [-3, 3] |
| `b` | float64 | 均线发散权重（X2系数） | 0.3 | [-3, 3] |
| `c` | float64 | 动量权重（X3系数） | 0.1 | [-3, 3] |
| `beta` | float64 | Sigmoid 激进系数 | 1.0 | [0.1, 5.0] |
| `gamma` | float64 | 仓位偏置系数（0=纯信号驱动） | 0.3 | [0, 2.0] |
| `sigma_floor` | float64 | 波动率最小值（防除零） | 0.005 | [0.001, 0.05] |
| `macro_interval_days` | int | 基础 DCA 间隔天数 | 30 | [7, 90] |
| `macro_accel_threshold` | float64 | 价格偏离触发加速阈值 | 0.05 | [0.01, 0.20] |
| `macro_accel_multiplier` | float64 | 加速倍率 | 2.0 | [1.0, 5.0] |
| `soft_release_months` | int | 软释放老化月数 | 6 | [1, 24] |
| `max_soft_release_pct` | float64 | 最大软释放比例 | 0.30 | [0.05, 0.50] |
| `dead_hold_target` | float64 | 目标底仓比例 | 0.50 | [0.10, 0.90] |
| `micro_reserve_pct` | float64 | 微观层资金保留比例 | 0.25 | [0.05, 0.50] |
| `ema_short_bars` | int | 短周期均线窗长 | 21 | [5, 55] |
| `ema_long_bars` | int | 长周期均线窗长 | 55 | [20, 200] |

### 7.2 结构约束

```
ema_short_bars < ema_long_bars                    // EMA 顺序约束
macro_interval_days >= 7                          // DCA 间隔最小值
soft_release_months >= 1                          // 释放老化至少 1 个月
dead_hold_target + micro_reserve_pct <= 0.95      // 避免过度预留
```

## 8. StrategyInput / StrategyOutput 契约

### 8.1 StrategyInput

```go
StrategyInput {
    Closes          []float64    // 收盘价序列（已 ACL 降级，不含 OHLCV）

    CurrentPrice    float64      // 当前价格
    Portfolio       PortfolioSnapshot  // 账户快照
    Runtime         RuntimeState       // 策略运行时状态
    AISignalVector  AISignalVector     // AI 信号（可选，回测时为 [0,0,0]）
}
```

### 8.2 PortfolioSnapshot

```go
PortfolioSnapshot {
    CNYBalance       float64  // CNY 现金余额
    DeadHold         float64  // 宏观底仓（份额）
    FloatHold        float64  // 微观浮动仓（份额）
    ColdSealedHold   float64  // 冷封存（份额）
    TotalEquity      float64  // 总权益（CNY 估值）
    DeadHoldLots     []SpotLot  // 底仓 lot 明细
}
```

### 8.3 RuntimeState

```go
RuntimeState {
    LastProcessedBar    int64    // 最后处理 bar 的时间戳
    LastMacroAction     int64    // 上次宏观操作时间戳
    AccumulatedReserve  float64  // 累计预留资金
}
```

### 8.4 StrategyOutput

```go
StrategyOutput {
    MacroIntent     *MacroIntent    // 宏观引擎意图（nil = 无操作）
    MicroIntent     *MicroIntent    // 微观引擎意图（nil = 无操作）
    ReleaseIntent   *ReleaseIntent  // 底仓释放意图（nil = 无操作）

```

### 8.5 Intent 子结构

```go
MacroIntent {
    Action      string   // "BUY"

    Action      string   // "BUY" | "SELL"
```

## 9. Step() 函数伪代码

```
func Step(input StrategyInput, params Params) StrategyOutput:

    // 1. 数据充足性检查
    if len(input.Closes) < max(params.EMAShortBars, params.EMALongBars, 21):
        return emptyOutput()

    // 2. 计算核心指标
    totalEquity = input.Portfolio.TotalEquity
    spendableCNY = max(0, input.Portfolio.CNYBalance - totalEquity × params.MicroReservePct)
    currentMicroWeight = input.Portfolio.FloatHold × input.CurrentPrice / totalEquity

    // 3. 市场状态感知
    marketState = ComputeMarketState(input.Closes, input.Timestamps)

    // 4. 宏观引擎
    macroIntent = ComputeMacroDecision(MacroInput{
        totalEquity, spendableCNY, input.CurrentPrice,
        input.Portfolio.DeadHold, input.Portfolio.DeadHold × input.CurrentPrice,
        params, marketState, input.Runtime,
    })

    // 5. 微观引擎
    microIntent = ComputeMicroDecision(MicroInput{
        input.Closes, input.CurrentPrice, totalEquity,
        currentMicroWeight, params, marketState,
    })

    // 6. 释放规则
    releaseIntent = ComputeDeadRelease(ReleaseInput{
        input.Portfolio.DeadHoldLots, params, input.Runtime, microIntent,
    })

    // 7. 更新 RuntimeState
    newRuntime = input.Runtime
    newRuntime.LastProcessedBar = input.Timestamps[len(input.Timestamps)-1]
    if macroIntent != nil:
        newRuntime.LastMacroAction = input.Timestamps[len(input.Timestamps)-1]

    // 8. 组装返回
    return StrategyOutput{macroIntent, microIntent, releaseIntent, marketState, newRuntime}
```
